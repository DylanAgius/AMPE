If you encounter problems while building and/or using one of the SUNDIALS solvers, we suggest you try to:
  1.check the FAQ and consult the Usage Notes section.
  2.verify whether this is a known issue and a patch is provided.
  3.browse the sundials-users mailing list archive.
  4.subscribe and post to the mailing list.
  5.report a bug through Bugzilla.


FAQ and Usage Notes

Answers to some of the most common user questions, as well as several tips on installing and using SUNDIALS can be found in the Frequently Asked Questions section.

Some lenghtier discussions of various issues are available in the Usage Notes section. Topics currently covered include: integration return and output times, controlling unphysical solutions and integrating over discontinuities, and switching on/off forward sensitivity computations.

Known Issues

Workarounds and patches for known bugs in SUNDIALS can be found in the Patches section.

Mailing Lists

We maintain two mailing lists, one (sundials-announce) reserved for announcements of new releases, the other (sundals-users) devoted to user-developer and user-user interaction. Previous posts to the sundials-user mailing list are archived and available for browsing.

To subscribe to one or both mailing lists, use the following form:

Selection: sundials-announce (sundials-announce@llnl.gov)
Low-volume, moderated list for announcements of new SUNDIALS releases.
It cannot be posted to by users.
sundials-users (sundials-users@llnl.gov)
Unmoderated discussion group for user-user and user-developer discussions.
It can be posted to by any member of the list.

Bugzilla Bug Tracking

SUNDIALS uses a system called Bugzilla to keep track of bugs. Bugzilla is easy to use and makes it easy to see how your bug has progressed, and when it is resolved.

To report a bug, you first must create a Bugzilla account (link will open in new window). This is easy and only requires a valid email address. Once you create your account, a password will be mailed to the email address you specified. You only need to create an account once, and you can use it from then on out to report as many bugs as you can find.

Once you receive the password, you can enter a new bug report (new window).
